# WeBro FM Admin Panel — React UI Prompt

## Overview
Create a complete admin panel UI in React + TypeScript + Vite + Tailwind CSS + shadcn/ui. Dark theme only (#0f172a background, Electric Blue #00D2FF, Purple #7000FF). This is a testing panel — must have every API endpoint connected and working.

---

## 1. Authentication (Two methods)

### Method A: OTP Login (Regular Users)
```
POST /api/auth/send-otp/
Body: { "phone_number": "9999999999" }
→ 200: { "message": "OTP sent successfully", "otp": "123456" }
(Dev mode OTP is always 123456 for phone 9999999999)

POST /api/auth/verify-otp/
Body: { "phone_number": "9999999999", "otp_code": "123456" }
→ 200: { "access": "jwt_token", "refresh": "jwt_token", "user": { id, full_name, phone_number, email } }
```

### Method B: Password Login (Admin Users)
```
POST /api/admin/login/
Body: { "phone_number": "8888888888", "password": "Admin@12345" }
→ 200: { "access": "jwt_token", "refresh": "jwt_token", "user": { id, full_name, phone_number, is_superuser } }
```

### Dev dummy OTP credentials
- Phone: `9999999999`
- OTP: `123456`

### Admin credentials
- Phone: `8888888888`
- Password: `Admin@12345`

## 4. Technical Requirements

### Auth Flow
- Store `access_token` and `refresh_token` in localStorage
- Create an Axios/fetch interceptor: on 401, call `POST /api/auth/refresh/` with `{ refresh: refreshToken }`, get new access token, retry request
- On refresh failure, redirect to login
- Base URL: `https://api.webrofm.in`

### Theme
- Background: `#0f172a` (slate-900)
- Primary: Electric Blue `#00D2FF`
- Accent: Purple `#7000FF`
- Cards: `#1e293b` (slate-800)
- Text: `#f1f5f9` (slate-100)
- Muted text: `#64748b` (slate-500)
- Borders: `#334155` (slate-700)
- Use shadcn/ui dark theme variant

### State Management
- React Query (TanStack Query) for all API calls with caching
- OR simple fetch with useState/useEffect (no external state lib needed for testing)

### Form Handling
- React Hook Form + Zod for validation
- File upload preview for images
- Loading states (spinners/skeleton)
- Toast notifications for success/error

### Routing
- React Router v6
- Protected routes (redirect to login if no token)
- /login → LoginPage
- / → redirect to /dashboard
- /dashboard → DashboardPage
- /users, /categories, /authors, /shows, /episodes, /comments, /reports, /feedback, /stories, /sponsorships, /storage, /likes, /favorites

### Layout
- Fixed sidebar (left, 240px) with scrollable nav
- Top header bar (shows current page title + breadcrumb)
- Main content area (padding, scroll)
- Sidebar has icons + labels, collapses to icons-only on smaller screens

### Data Display
- Tables with: sortable columns, pagination (20 per page), search bar where applicable
- All tables show total count
- Empty state message when no data
- Confirm dialogs for delete actions (shadcn AlertDialog)

## 5. Example API Call Pattern (Add this to all pages)

```typescript
const API_BASE = 'https://api.webrofm.in/api';

// GET with auth
const res = await fetch(`${API_BASE}/admin/users/`, {
  headers: { Authorization: `Bearer ${token}` }
});

// POST with auth + JSON
const res = await fetch(`${API_BASE}/admin/users/<id>/action/`, {
  method: 'POST',
  headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' },
  body: JSON.stringify({ action: 'block' })
});

// POST with auth + file upload
const formData = new FormData();
formData.append('name', 'New Category');
formData.append('slug', 'new-category');
formData.append('icon', file);
const res = await fetch(`${API_BASE}/admin/categories/`, {
  method: 'POST',
  headers: { Authorization: `Bearer ${token}` },
  body: formData
});
```
