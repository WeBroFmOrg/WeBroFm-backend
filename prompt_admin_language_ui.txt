## 2. API Endpoints for Languages

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/admin/languages/` | List all languages |
| POST | `/api/admin/languages/` | Create new language |
| GET | `/api/admin/languages/{id}/` | Get language detail |
| PATCH | `/api/admin/languages/{id}/` | Update language |
| DELETE | `/api/admin/languages/{id}/` | Delete language |

**Language object structure:**
```json
{
  "id": 1,
  "name": "English",
  "code": "en",
  "icon": "https://...",  // ImageField, optional
  "is_active": true
}
```

**POST/PATCH body:**
```json
{
  "name": "English",
  "code": "en",
  "icon": <file>,      // optional image upload
  "is_active": true
}
```

## 3. Languages Page — Layout

### Tab: All Languages (default)
Table with columns:
| # | Name | Code | Icon | Status | Actions |
|---|------|------|------|--------|---------|

- **Icon**: small preview image (32x32), fallback to language code text badge if no icon
- **Status**: green chip "Active" or gray chip "Inactive"
- **Actions**: ✏️ Edit | 🗑️ Delete

### Tab: Create Language
Form fields:
- **Name** (required, text) — e.g. "English", "Hindi", "GenZ"
- **Code** (required, text) — short code like `en`, `hi`, `genz`, `mr`, `gu`, `ta`, `te`, `bn`
- **Icon** (optional, image upload) — small flag/icon for the language
- **Active** (toggle switch, default ON)

Form submit → POST `/api/admin/languages/` (multipart/form-data for icon upload)

### Edit Language (modal or drawer)
Same form as create but pre-filled. PATCH `/api/admin/languages/{id}/`.

### Delete Language
Confirmation dialog → DELETE `/api/admin/languages/{id}/`
