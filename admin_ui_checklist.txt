WEBRO FM ADMIN UI - CHECKUP PROMPT
====================================

Is prompt ko use karke check karo ki jo UI bana hai woh sahi hai ya nahi.
Har page ke liye features diye hain — agar sab match karta hai to UI sahi hai.

────────────────────────────────────────────
TECH STACK
────────────────────────────────────────────
React + TypeScript + Vite + Tailwind CSS
shadcn/ui ya NextUI (component library)
react-router-dom v6 (routing)
axios (API calls)
@tanstack/react-query (server state)
recharts ya chart.js (charts)
react-hook-form + zod (forms)
lucide-react (icons)
sonner ya react-hot-toast (notifications)

Frontend Build Output → Django ke staticfiles folder mein ya alag serve karo

────────────────────────────────────────────
COLOR THEME
────────────────────────────────────────────
Dark theme ONLY (no light mode toggle)
Background: #0f172a (slate-900) / #020617 (slate-950)
Cards: glass effect (backdrop-blur, rgba backgrounds)
Primary: Electric Blue #00D2FF
Secondary: Accent Purple #7000FF
Success: Green #53E076
Text: white/off-white, slate-400 for muted
Active sidebar: gradient (blue -> purple)
All gradients use blue->purple combo

────────────────────────────────────────────
LAYOUT
────────────────────────────────────────────
Left sidebar (collapsible on mobile via hamburger):
  - Logo + "Webro FM Admin" at top
  - Sidebar items with icons:
    * Dashboard
    * Users
    * Content (expandable): Shows, Episodes, Categories, Authors
    * Engagement (expandable): Comments, Likes, Favorites
    * Feedback & Reports (expandable): Feedback, Reports
    * Collaboration (expandable): Stories, Sponsorships
    * Storage → R2 Explorer
  - Active page: gradient highlight (blue->purple)
  - Bottom: Admin name + Logout button

Top header bar:
  - Hamburger menu (mobile only)
  - Current page title
  - Admin avatar/name dropdown (with logout)

Breadcrumbs optional but nice

PAGES - CHECKLIST
────────────────────────────────────────────

--- 1. LOGIN PAGE ---
  Route: /admin/login
  API: POST /api/admin/login/
  Body: { phone_number, password }
  Response: { access, refresh, user: { id, full_name, phone_number, is_superuser } }

  Check:
  [ ] Dark full-screen background with gradient glow
  [ ] Centered login card (max-w-md), glass effect
  [ ] Brand logo/icon at top
  [ ] Phone number input
  [ ] Password input with show/hide toggle
  [ ] "Sign In" button with loading state
  [ ] Error toast on invalid credentials
  [ ] On success: tokens store in localStorage, redirect to /admin/dashboard
  [ ] Auto-redirect to dashboard if already logged in
  [ ] Axios interceptor: attach Bearer token to all requests
  [ ] Axios interceptor: on 401 → try refresh token → if fail → redirect to login

--- 2. DASHBOARD ---
  Route: /admin/dashboard
  API: GET /api/admin/dashboard/stats/

  Check:
  [ ] Stats cards grid (2x2 mobile, 4x4 desktop): Users, Shows, Episodes, Hits, Pending Stories, Pending Ads
  [ ] Each card: glass effect, count number, icon, subtitle

  [ ] Line chart: Listener Activity (last 7 days)
  [ ] Doughnut/pie chart: Shows by Category
